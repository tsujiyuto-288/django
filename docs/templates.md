# テンプレートについて

## 変数の表示（{{ }} について）

ビュー（`views.py`）から渡された変数やオブジェクトのデータを、HTML上に展開して表示する。

### 💡 概要

- `{{ 変数名 }}` のように、変数を二重の中括弧で囲むことで、その中身を文字列としてWebページに出力できる。
- オブジェクトの属性、辞書のキー名、リストのインデックスなどにアクセスする場合は、Pythonのようなブラケット `[]` ではなく、すべて **ドット（`.`）** で繋ぐ。

### ✒️ 基本的な書き方

```html
<!-- 1. オブジェクトの属性（プロパティ）にアクセスする場合 -->
<!-- Pythonでの book.title と同じようにドットで繋ぐ -->
{{ book.title }}

<!-- 2. リスト（配列）やクエリセットのN番目にアクセスする場合 -->
<!-- Pythonでの book_dict[2].title ではなく、[ ]は使わずドット(.)の後にインデックス（数字）を書く -->
{{ book_dict.2.title }}

<!-- （参考）辞書（dict）のキーにアクセスする場合も同様 -->
<!-- Pythonでの data["name"] ではなく、ドットで繋ぐ -->
{{ data.name }}
```

## フィルタ（|）

変数を出力する際に、値の見た目やフォーマットなどを変換・加工するための機能。

### 💡 概要

- `{{ 変数名|フィルター名 }}` のように、変数の後にパイプライン（`|`）をつなぐことで適用できる。
- フィルターによっては `:` に続けて引数（追加の指定や条件）を渡すことができる。

### ✒️ 基本的な書き方

```html
<!-- 1. default: 変数が空（存在しない、またはNoneなど）の場合に表示する値を指定する -->
<!-- 例: book_stockが空の場合「在庫がありません」と表示される -->
{{ book_stock|default:"在庫がありません" }}

<!-- 2. date: 日付データの表示形式をフォーマットする -->
<!-- Yは4桁の年、mは2桁の月、dは2桁の日を表す -->
{{ today|date:"Y年m月d日" }}

<!-- 3. length: リストの要素数や、文字列の文字数をカウントして表示する -->
<!-- 引数は取らないため : は不要 -->
{{ book.title|length }}

<!-- 4. linebreaksbr: 変数内の改行文字（\nなど）を、HTMLの改行（<br>タグ）に変換する -->
{{ book.label|linebreaksbr }}

<!-- 5. truncatechars_html: 指定した文字数で文字列を切り詰め、超えたぶんの末尾に「…」を追加する -->
<!-- HTMLタグを考慮・維持しつつ、文字数を制限（今回は10文字）する -->
{{ book.label|truncatechars_html:10 }}
```

## {% %} テンプレートタグ

条件分岐や繰り返しなど、処理的なことをするときに使用する。

### for（繰り返し）

- Pythonの `for` 文と同様に、イテラブルからデータを1つずつ取り出して処理を繰り返す。
- 最後は `{% endfor %}` で閉じる。

```html
<!-- book_dictからデータを1つずつ取り出し、変数bookに入れて展開を繰り返す -->
{% for book in book_dict %}
<div class="card-body">
  <p class="mb-0">タイトル：{{ book.title }}</p>
  <p class="mb-0">値段：{{ book.price }}</p>
</div>
{% endfor %}
```

### if（条件分岐）

- Pythonの `if` 文と同様に、指定した条件に応じて内容を分岐させる。
- 最後は `{% endif %}` で閉じる。
  ※Pythonとは異なり、カッコ `()` を使った複雑な条件のグループ化や、計算、メソッドの呼び出しなど難しいことはできない。

```html
<!-- 変数の値によって出力するHTMLやテキストを出し分ける -->
{% if book.status == "order" %}
<p>未発注の状態です</p>
{% elif book.status == "waiting" %}
<p>発注済みですが、まだ陳列されていません。</p>
{% else %}
<p>陳列済みです。</p>
{% endif %}
```

## テンプレートの継承（extends / block）

すべてのページで共通するレイアウト枠組み（メニューやヘッダなど）を1つの親ファイルで管理し、各ページでは「ページ固有の中身だけ」を差し替えて表示するための仕組み。

### 💡 概要

- 画面上部のナビゲーションメニューや、決まりきったJSの読み込みを親（ベース）のファイルで一元管理できるため、保守性が非常に高くなる。
- 使用するときは、必ずファイルの1番最初の行に `{% extends "ファイル名" %}` を記述する。

### ✒️ 基本的な書き方

**1. 親となるベースファイルの準備（例: `test_template.html`）**
親となるHTMLファイル内に、中身を差し替えたい場所を示す「空箱（ブロック）」を用意する。

```html
<!DOCTYPE html>
<html lang="ja">
  <body>
    <!-- 全ページ共通のナビゲーションやメニュー等 -->
    <h1>共通メニュー</h1>

    <!-- この箱（ブロック）の中に子ページのhtmlが流し込まれるイメージ -->
    {% block content %} {% endblock %}

    <!-- ブロックは複数定義可能。ページ固有のJS用枠などを用意しておくと安全 -->
    {% block js_content %} {% endblock %}
  </body>
</html>
```

**2. 子となる個別ページの書き方（例: `test2.html`）**

```html
<!-- 必ずファイルの1行目に記述する -->
{% extends "test_template.html" %}

<!-- 親側の content ブロックにおさまる部分（見た目）を書く -->
{% block content %}
<p>このページ固有のメインコンテンツをここに書きます。</p>
{% endblock %}

<!-- 親側の js_content ブロックにおさまる部分（裏側の処理）を書く -->
{% block js_content %}
<script>
  console.log("このページだけのJS処理");
</script>
{% endblock %}
```

## includeについて

別のHTMLファイルの内容を、その場所へそっくりそのまま埋め込むための仕組み。

### 💡 概要

- 何回も使いまわせるので便利。
- 使い回すので保守性が上がる。

### ✒️ 基本的な書き方

**1. 埋め込まれる側の部品ファイル（例: `modal.html`）**
呼び出される側のファイルに `<html>` などの骨組みは不要。純粋に展開したいHTMLだけを記述する。

```html
<!-- modal.html -->
<div class="custom-modal">
  <p>ここにモーダルやカードなどの部品のコードを書く</p>
</div>
```

**2. 呼び出す側のファイル（例: `test3.html`）**
部品を表示させたい位置に `{% include "ファイル名" %}` を記述するだけで、その場所に展開される。

```html
<h1>メインコンテンツ</h1>
<!-- ここに modal.html の内容がそのまま挿入される -->
{% include "modal.html" %}
```
