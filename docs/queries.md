# クエリセットについて

## クエリセット (QuerySet) とは

データベースから取り出したデータの「まとまり」を管理するための、Django専用の**特製ボックス（リストの強化版）**。

### 💡 なぜただのリストではなく「クエリセット」なのか？

1. **色々な機能（メソッド）を繋げ続けられるから**
   ただのリストにしてしまうと、そこで処理が完結してしまう。クエリセットの箱のままであれば、`Book.objects.filter(...).order_by(...).exclude(...)` のように、後ろに条件を無限に繋げていくことができる。
2. **遅延評価（ギリギリまでDBを見に行かない）のため**
   クエリセットは、データそのものではなく「こういう条件でデータを取る予定です」という**予約チケット**のような状態。
   実際にデータが必要になる（`list()`で変換されたり、`for`文でループされたりする）限界ギリギリの瞬間までデータベースにアクセスしないため、無駄な通信が減り動作が速くなる。

### 💡 「外側がクエリセット」のイメージ

Djangoはデータベースからデータを持ってくる際、必ずこの「クエリセット」という特製の箱に入れて届けてくれる。

- **通常時**: クエリセット（特製の箱）の中に、モデルオブジェクトが入っている。
- **`.values()` 使用時**: クエリセット（特製の箱）の中に、辞書（dict）が入っている。

Javascript等にJSONとしてデータを送る場合、「Django特製の箱」のままではエラーになる。そのため、最後に `list()` で囲むことによって「箱から中身を取り出し、純粋なPythonの配列（リスト）に戻す」という処理を行っている。

---

## クエリセットを辞書やリストに変換する方法

### 💡 内容

- **`.values()`**: 中身を **辞書型 (dict)** にして抽出する。
- **`.values_list()`**: キーを無視して **値だけ（リスト型）** を抽出する。
- どちらも抽出直後の大枠はクエリセットのため、素のデータにするには `list()` で囲む必要がある。
- どちらも引数でカラム名を指定すること指定したカラムのデータだけを取得してくれる

### ✒️ 基本的な書き方

```python
# --- .values() ---
# すべてのカラムを辞書化  [{"id": 1, "title": "本"}, {"id": 2, "title": "本2"}]
books = list(Book.objects.filter(author__name="太宰").values())

# 欲しいカラムだけ指定  [{"title": "本"}]
books_only_title = list(Book.objects.filter(author__name="太宰").values("title"))


# --- .values_list() ---
# 基本はタプルのリストになる  [("本1",), ("本2",)]
books_tuple = list(Book.objects.filter(author__name="太宰").values_list("title"))

# flat=True をつけるとリストになる  ["本1", "本2"]
books_flat = list(Book.objects.filter(author__name="太宰").values_list("title", flat=True))
```

## get() について

データベースから **「条件に完全に一致する1つのデータ」** を引き当ててくるメソッド

### 💡 内容

- **モデルオブジェクトが返ってくる。** そのため、取得してすぐに `book.title` のようにドットで繋いで各種データを取り出せる
- `pk=1` や `title="銀河鉄道の夜"` などの条件を、カッコ内に直接指定して検索できる
- **結果が「1件」じゃないとエラーになる**ので注意

### ✒️ 基本的な書き方

```python
# 基本的な書き方
book = Book.objects.get(pk=1)
# オブジェクトが直接返ってくるので、すぐ属性にアクセスできる
book_title = book.title
book_price = book.price

# 取得したいものが決まっている場合はそもまま.を繋げてかける。
book = Book.objects.get(pk=1).title
```

## filter() について

データベースから **「条件に一致する複数のデータ」** を引き当ててくるメソッド。

### 💡 内容

- **「クエリセット」の箱につつまれて返ってくる。** そのため、条件に合うものが1件しかない場合や、0件（見つからなかった）の場合でも、必ずクエリセットの形で返ってくる。
- 引数には `status="order"` のように検索したい条件を指定する。カンマ（`,`）で複数つなげると、AND検索になる。
- **1対多や多対多のリレーション用フィールド（外部キー等）を条件にする場合**は、 **「関連モデルのID（数字）」** か **「関連モデルのオブジェクトそのもの」** を渡す必要がある。

### ✒️ 基本的な書き方

```python
# --- 基本的な書き方 ---
# 返り値は必ずクエリセットになる
books = Book.objects.filter(status="order")

# カンマで区切るとAND検索になる
books = Book.objects.filter(status="order", price=1000)

# --- リレーション先を指定した検索（1対多・多対多のフィールド） ---

# パターン1: ID（数字）を直接渡して検索する
company_id = 1
books = Book.objects.filter(company=company_id)

# パターン2: オブジェクトそのものを渡して検索する
company_obj = Company.objects.get(pk=1)
books = Book.objects.filter(company=company_obj)
```

## \_\_によるリレーションの参照(ルックアップ)

外部キー等で関連付けられている別テーブル（他のモデル）のデータをまたいで検索・参照するための記法。

### 💡 概要

- 逆参照であっても(元のモデル名の小文字)でいい。\_setなど不要。

### ✒️ 基本的な書き方

```python
# 「著者(author)」の「名前(name)」が一致する本を検索する
books = Book.objects.filter(author__name="太宰")
# 「著者」の「スポンサー(sponsor)」の「会社名(company_name)」で検索
books_by_sponsor = Book.objects.filter(author__sponsor__company_name="〇〇企業")
# 本のタイトルから著者を検索（逆参照）
authors = Author.objects.filter(book__title="本のタイトル")
```

## \_\_による検索条件の指定（フィールドルックアップ）

`フィールド名__条件` と繋ぐことで、完全一致以外の複雑な絞り込み（以上、未満、部分一致など）を行うことができる

### 💡 よく使う条件一覧

| 条件                          | 意味                         | 備考                         |
| ----------------------------- | ---------------------------- | ---------------------------- |
| `__contains`                  | 部分一致（「〜を含む」）     | 大文字と小文字を`区別する`   |
| `__icontains`                 | 部分一致（「〜を含む」）     | 大文字と小文字を`区別しない` |
| `__in`                        | リストの中のどれかに一致する |                              |
| `__gt`                        | より大きい（＞）             | Greater Thanの略             |
| `__gte`                       | 以上（≧）                    | Greater Than or Equalの略    |
| `__lt`                        | より小さい（＜）             | Less Thanの略                |
| `__lte`                       | 以下（≦）                    | Less Than or Equalの略       |
| `__startswith` / `__endswith` | 〜で始まる / 〜で終わる      |                              |

### ✒️ 基本的な書き方

```python
# 部分一致（タイトルに「魔法」が含まれる本を検索）
books = Book.objects.filter(title__contains="魔法")

# in検索：リストにある値のどれかに一致する（IDが 1 か 3 か 5）
books = Book.objects.filter(id__in=[1, 3, 5])

# 比較検索：以上・以下（値段が 1000円 以上の本）
books = Book.objects.filter(price__gte=1000)

# 著者の名前に「太宰」が含まれる本
books = Book.objects.filter(author__name__contains="太宰")
```

## AND検索（複数条件の指定）

「A かつ B」という、複数の検索条件をすべて満たすデータを引き当ててくる記法。

### 💡 概要

- filter カッコ内で、条件をカンマ（`,`）で区切って並べる
- または、filter().filter() のように直列で繋げていく方式（メソッドチェーン）でも全く同じ結果になり、AND条件として扱われる。

### ✒️ 基本的な書き方

```python
# パターン1: カンマで区切る
# 「著者の名前に"太宰"が含まれる」 かつ 「出版社の名前に"集英社"が含まれる」 本
books = Book.objects.filter(
    author__name__contains="太宰",
    company__name__contains="集英社"
)

# パターン2: さらに後ろに繋げる（メソッドチェーン）
# Pythonのif文などで、後から「条件を追加していきたい時」などに便利
books = Book.objects.filter(author__name__contains="太宰").filter(company__name__contains="集英社")
```

## OR検索（Qオブジェクト）

A or B　という書き方。

### 💡 概要

- `Q()`で囲み`|`を使って書くことでor検索ができる。
- Qを使用するためにはimportする必要がある。
- なんでQで加工必要があるかは一旦知らなくていい。そういうものという理解で。

### ✒️ 基本的な書き方

```python
# Qのインポート
from django.db.models import Q

# 「著者の名前が"太宰"」 または 「出版社の名前が"集英社"」 に一致する本
books = Book.objects.filter(
    Q(author__name="太宰") | Q(company__name="集英社")
)

# 3つ以上の条件を繋ぐ場合も同じように | で繋げていく
books = Book.objects.filter(
    Q(author__name="太宰") | Q(company__name="集英社") | Q(title__contains="走れ")
)
```

## 不等号を用いた検索

### 💡 概要

- `フィールド名__条件` の形で繋ぐことで、比較検索を行うことができる。

| 条件      | 意味                         | 備考                       |
| --------- | ---------------------------- | -------------------------- |
| `__gte`   | 以上（≧）                    | Greater Than or Equal の略 |
| `__gt`    | より大きい / 超（＞）        | Greater Than の略          |
| `__lte`   | 以下（≦）                    | Less Than or Equal の略    |
| `__lt`    | より小さい / 未満（＜）      | Less Than の略             |
| `__in`    | 指定したリストのどれかに一致 | 3つ以上指定することも可能  |
| `__range` | 指定した範囲に一致           | startとendの値も含む       |

### ✒️ 基本的な書き方

```python
# 以上の場合
books = Book.objects.filter(price__gte=price)

# 超（より大きい）の場合
books = Book.objects.filter(price__gt=price)

# 以下の場合
books = Book.objects.filter(price__lte=price)

# 未満（より小さい）の場合
books = Book.objects.filter(price__lt=price)

# 指定した値段のどちらか（price_1 または price_2）に完全一致する本を検索
books = Book.objects.filter(price__in=[price_1, price_2])

# 3つ以上の値を指定することも可能（IDが 1、3、5 のどれかに一致する本）
books = Book.objects.filter(id__in=[1, 3, 5])

# 範囲指定(100~500円の間。100と500も含む)
Book.objects.filter(price__range=[100, 500])
```

## よくつかうメソッド

### 💡 概要

| メソッド名 | 戻り値                       | 役割                                                                           | 注意点                                                                                       |
| ---------- | ---------------------------- | ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------- |
| `exists()` | `True` または `False`        | データが1件でも存在するかを判定する。データの有無だけを知りたい時につかうといい。      | 引数で検索できない                                                                           |
| `count()`  | 数値（整数）                 | データが何件あるかを数える。                                                   | 引数で検索できない                                                                           |
| `save()`   | なし（`None`）               | 作成や変更を行った単一のデータ（インスタンス）をデータベースに保存・更新する。 | クエリセット（`filter`の結果など）に対しては使えない。1つのデータに対してのみ使用可能。      |
| `delete()` | 削除した件数の詳細（タプル） | 対象のデータをデータベースから完全に削除する。                                 | 単一のデータだけでなく、`filter()`などのクエリセットに対して使い「一括削除」することも可能。 |
